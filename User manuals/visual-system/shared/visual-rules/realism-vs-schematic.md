# Realism vs Schematic

## Classification policy
Choose the image class by instructional need.

| Class | Use for | Primary goal |
|---|---|---|
| realistic | hero, installed context, identity shots, fingerprint use, app pairing context | preserve product identity in believable use |
| semi-realistic | keypad use, language settings, troubleshooting gestures | keep interaction clear while simplifying noise |
| schematic | installation, wiring, anatomy, mounting, reset sequences, battery replacement | maximize clarity over realism |
| hybrid | flows that need a real product shell plus simplified overlays or callouts | preserve recognition while clarifying steps |

## Rule
If realism hides the step, switch classification.
If schematic treatment risks losing the product family, reintroduce the real outer geometry as a hybrid.
