# AI Limitations

## Known failure modes
- generated text and numbers are unreliable
- app interfaces drift quickly into fiction
- black products lose edge definition under bad lighting
- hands and fingers deform under tight interaction scenes
- model bias can replace a specific lock with a generic smart-lock silhouette

## Operational response
- do not use generated text as instructional proof
- keep app UI low-detail unless using real captures
- validate every output against the visual profile before approving it
- use schematic fallback for wiring, installation, and anatomy flows
- review product families side by side to catch cross-product contamination
