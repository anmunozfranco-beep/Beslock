# App UI Policy

## Default rule
Real app captures are preferred over generated UI for any image that must explain actual screens, labels, buttons, or settings.

## Acceptable AI use
Generated phone imagery is acceptable only when the phone screen is contextual support and not the source of truth for the instruction.

## When AI UI is not acceptable
- step-by-step navigation
- exact button labels
- menu hierarchy explanations
- troubleshooting based on specific on-screen errors

## Recommended split
- use real captures for UI detail
- use generated contextual imagery for pairing atmosphere only
