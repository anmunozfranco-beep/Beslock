# Reusable Assets

## Reusable interaction families
- fingerprint interaction framing
- keypad interaction framing
- app pairing context
- QR pairing context
- troubleshooting gesture language
- downloads and documentation still-life compositions

## Reuse rule
Reuse the scene logic, not the product geometry.
A reusable asset is acceptable only if product-specific identity is either:
- intentionally abstracted, or
- replaced with the correct product shell before publication

## Strong reuse candidates
- Smart Life onboarding context shots
- QR pairing atmosphere shots
- generic downloads or documentation support visuals
- neutral troubleshooting body language references

## Weak reuse candidates
- hero product identity shots
- any scene where the fingerprint sensor location matters
- any scene where keypad layout is product-specific
- any scene that depends on exact handle, knob, or rim-lock geometry
