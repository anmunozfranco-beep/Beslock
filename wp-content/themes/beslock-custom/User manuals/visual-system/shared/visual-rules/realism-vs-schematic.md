# Realism vs Schematic

## Classification policy
Choose the image class by instructional need.

For manual production after the 2026-05-13 reset, only `schematic` and `hybrid` are valid target classes.
`realistic` and `semi-realistic` remain in the taxonomy only to help interpret legacy files.

| Class | Use for | Primary goal |
|---|---|---|
| realistic | legacy-only historical category | interpret older production records; do not use for new manual work |
| semi-realistic | legacy-only historical category | interpret older production records; do not use for new manual work |
| schematic | installation, wiring, anatomy, mounting, reset sequences, battery replacement, keypad or fingerprint instruction, troubleshooting | maximize clarity and side control over realism |
| hybrid | exterior identity, installed context, app context, documentation support, product shell plus simplified cues | preserve recognition while keeping the image manual-safe |

## Rule
If manual clarity conflicts with realism, abandon realism.
If schematic treatment risks losing the product family, reintroduce only the true outer geometry as a hybrid.
Do not escalate back toward photorealistic scenes for manual work once the reset policy is active.
