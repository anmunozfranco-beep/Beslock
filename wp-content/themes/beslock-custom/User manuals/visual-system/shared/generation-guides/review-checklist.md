# Review Checklist

## Pass conditions
- [ ] The product is still unmistakably the intended Beslock model.
- [ ] Protected geometry matches the visual profile.
- [ ] The interaction zone is visible and correctly used.
- [ ] The chosen class is `schematic` or `hybrid` and fits the instructional goal.
- [ ] Lighting or shading preserves black-surface readability without forcing photo realism.
- [ ] Any hand cue is limited to a simplified hand-only silhouette when truly needed.
- [ ] No full human figure, face, or body-led storytelling is present.
- [ ] The image shows only one hardware side at a time: exterior, interior, or edge.
- [ ] If the door edge or lock body appears, the mechanism matches the OEM-backed mechanism chosen for that product package.
- [ ] No generated text is carrying technical meaning.
- [ ] The asset can be assigned to a stable filename and manual slot.

## Reject when
- [ ] another product's geometry leaks in
- [ ] the lock family changes
- [ ] the key interaction area is hidden or wrong
- [ ] the image is visually attractive but not instructional
- [ ] a real-looking person or face becomes part of the explanation
- [ ] interior and exterior hardware appear together in a single manual frame
- [ ] the edge mechanism changes from one slot to another without OEM support
