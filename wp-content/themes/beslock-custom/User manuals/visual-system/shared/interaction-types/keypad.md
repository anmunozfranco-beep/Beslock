# Keypad Interaction

## Use when
The image supports PIN entry, configuration, or keypad-led troubleshooting.

## Prompt fragment
Simplified manual hand silhouette with one finger approaching the real keypad area of the smart lock, one clear press action, keypad layout remains product-accurate, schematic or hybrid framing, neutral background, technically useful clarity.

## Rules
- Keep the keypad layout consistent with the reference product.
- Do not depend on generated numbers or labels being readable for instructions.
- Show the action, not fake UI precision.
- Use no full human body or face; only a hand cue when the step truly needs contact context.
- If a product has no keypad, do not force this interaction type.

## Avoid
- invented keypad zones
- redesigned icon layouts
- readable but wrong text as instruction evidence
- turning the keypad into a phone screen aesthetic
- photorealistic people or lifestyle staging
