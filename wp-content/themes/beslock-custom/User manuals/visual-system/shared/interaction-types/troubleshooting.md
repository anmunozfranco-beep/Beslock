# Troubleshooting Interaction

## Use when
The image needs to support a failure state, review step, retry step, or diagnostic article.

## Prompt fragment
Calm troubleshooting scene, user checking the lock or app with a neutral problem-solving gesture, focus on the failing interaction zone, realistic domestic environment, sober documentation style.

## Rules
- Keep emotional tone neutral and diagnostic.
- Make the product and failure context legible without dramatic acting.
- Prefer small cues of hesitation or review, not frustration theater.
- If the issue is mechanical or internal, use schematic or hybrid visuals instead of forcing realism.

## Avoid
- alarmist expressions
- danger styling
- giant fake error overlays
- visual clutter that obscures the cause being discussed
