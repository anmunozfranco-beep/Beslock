# QR Pairing

## Use when
The asset illustrates pairing between a lock and a phone via QR code.

## Prompt fragment
Hybrid manual composition with the smart lock and a simplified smartphone during QR pairing, generic QR code visible only as a contextual cue, lock and phone both readable, clean neutral background, sober manual style.

## Rules
- Treat the QR as contextual, not as a scannable technical artifact.
- Keep the phone interface secondary to the pairing concept.
- Use this only for products or manuals that actually mention QR pairing.
- Preserve the real lock geometry while framing both devices.
- Do not add a full person; use a hand silhouette only when needed to explain orientation.

## Avoid
- oversized fake UI text
- highly readable but incorrect QR instructions
- floating code elements or holographic effects
- photorealistic people
