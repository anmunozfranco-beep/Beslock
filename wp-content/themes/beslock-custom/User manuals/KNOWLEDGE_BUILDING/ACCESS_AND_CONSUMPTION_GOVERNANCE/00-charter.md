# 00 — Access & Consumption Charter

## Principles
1. Access exposes; it does not transform meaning.
2. Consumption is publication-neutral; rendering lives elsewhere.
3. Every bundle carries provenance; provenance-stripped delivery is forbidden.
4. Safety filters are non-negotiable defaults; opt-out requires audit trace.
5. Determinism is a contract: same inputs, same snapshot, same bundle.
6. Disambiguation is preferred over guessing; ambiguity surfaces, never hides.
7. Access layer is subordinate to knowledge-core, lifecycle, and validation governance.
8. Future systems inherit access contracts; access contracts never bend to satisfy a consumer.

## Hard guarantees
- Modeling-only. Builds no chatbots, APIs, frontends, PDFs, images, renderers.
- Read-only at runtime. Mutates no per-product knowledge.
- Subordinate. Cannot override knowledge-core, lifecycle, or validation.
- Deterministic. Same inputs + snapshot => same bundle.
