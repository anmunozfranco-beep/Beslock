# e-Orbit
## Guía de estilo editorial

## 1. Objetivo
Definir cómo debe escribirse el contenido de e-Orbit para mantener consistencia en manuales, centro de ayuda y materiales asociados.

---

## 2. Principios base
- escribir para ayudar a hacer
- priorizar claridad sobre formalismo
- explicar sin sonar técnico de más
- escribir para consulta rápida
- no exagerar certezas si hay información por validar

---

## 3. Tono recomendado
Debe sonar:
- claro
- útil
- cercano
- ordenado
- confiable
- práctico

No debe sonar:
- excesivamente promocional
- ambiguo
- rebuscado
- demasiado frío

---

## 4. Reglas clave
- usar siempre **e-Orbit**
- usar siempre **Smart Life**
- empezar pasos con verbo
- usar rutas consistentes
- marcar con prudencia lo no validado
- no repetir pasos dentro de recomendaciones

---

## 5. Estructura sugerida por página
1. qué vas a hacer
2. qué necesitas antes
3. pasos
4. recomendación o advertencia
5. qué validar después
6. problemas frecuentes
7. siguiente paso recomendado
