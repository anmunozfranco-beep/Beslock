# Workflow Ergonomics & Cognitive Load

Six ergonomic principles bounded by safety discipline; ergonomics never override safety.

## Ergonomic principles

- `cognitive-load-reduction` — show at most one decision per checkpoint; defer secondary information to expandable detail
- `operational-fatigue-prevention` — queues respect declared SLA tiers; no operator is auto-assigned beyond declared capacity ceiling
- `escalation-fatigue` — repeated escalations from the same trigger collapse into one tracked incident, not N notifications
- `reviewer-overload` — reviewer registry declares per-reviewer concurrent claim ceiling
- `ambiguity-minimization` — every screen states what is known, what is uncertain, and the next required action
- `guided-operational-sequencing` — checkpoints are presented in a single declared order; no parallel competing decision surfaces

## Discipline

- ergonomics never override safety predicates
- ergonomics never compress disclosure of irreversibility or uncertainty
- ergonomics never replace required dual-review with single-click flows
- ergonomics never auto-default approval; default is always 'pending operator decision'
