# e-Orbit
## Checklist de validación del manual

## 1. Objetivo
Este checklist sirve para validar la calidad editorial, técnica y visual del manual de e-Orbit antes de publicación o maquetación final.

---

## 2. Validación editorial
- [ ] El nombre **e-Orbit** se usa de forma consistente
- [ ] El nombre **Smart Life** se usa de forma consistente
- [ ] Los títulos son claros y orientados a acción
- [ ] Los pasos empiezan con verbo
- [ ] Las rutas de menú usan un formato consistente
- [ ] Las recomendaciones no repiten pasos
- [ ] Las advertencias están justificadas
- [ ] No hay párrafos innecesariamente densos
- [ ] El tono es claro, práctico y consistente

---

## 3. Validación técnica
- [ ] Se validó el flujo de agregar administrador
- [ ] Se validó el flujo de agregar usuario
- [ ] Se validó el registro de PIN
- [ ] Se validó el registro de huella
- [ ] Se validó el cambio de idioma
- [ ] Se revisó el uso de `888#`
- [ ] Se revisó el uso de `3009#`
- [ ] Se validó el flujo de agregar dispositivo en Smart Life
- [ ] Se validó la vinculación por QR
- [ ] Las incertidumbres técnicas quedaron marcadas de forma prudente

---

## 4. Validación visual
- [ ] Existe imagen general del producto
- [ ] Existe imagen contextual de uso
- [ ] Existe apoyo visual para agregar administrador
- [ ] Existe apoyo visual para PIN
- [ ] Existe apoyo visual para huella
- [ ] Existe apoyo visual para idioma
- [ ] Existe apoyo visual para agregar dispositivo
- [ ] Existe apoyo visual para QR
- [ ] Existe apoyo visual para troubleshooting de huella
- [ ] Existe apoyo visual para troubleshooting de app

---

## 5. Validación de estructura
- [ ] La introducción es clara
- [ ] El bloque de primeros pasos está presente
- [ ] Los flujos principales están incluidos
- [ ] Los problemas frecuentes están resumidos
- [ ] El cierre indica pendientes reales antes de publicación

---

## 6. Cierre
- [ ] El manual está listo para maquetación
- [ ] El manual está listo para publicación web
- [ ] El manual requiere ajustes adicionales
