L37

Evidence Intake

Twelve-step governed intake workflow. Every step is a proposal; nothing is committed automatically.

Reviewer authorization is required for promotion. The console only emits an intake-draft manifest.

Workflow

01
Upload document upload-document
Operator selects a local file. The console reads filename + size only.
02
Detect format detect-format
MIME / extension inferred deterministically; no content parsing.
03
Infer evidence class infer-evidence-class
Default class proposed from layer-32 evidence taxonomy; operator can override.
04
Infer product scope infer-product-scope
Suggested e-orbit/e-prime/e-flex/e-touch/e-shield/e-nova/portfolio-level.
05
Infer trust level infer-trust-level
Default trust tier from layer-33 trust rules; never auto-promoted.
06
Suggest routing destination suggest-routing
Layer-33 routing-policy lookup; operator confirms.
07
Suggest affected domains/products suggest-affected-domains
Layer-34 domain map + layer-35 identity links.
08
Operator re-evaluation operator-re-evaluation
Operator may revise any inferred field with a reason field.
09
Operator override operator-override
Override is recorded as part of the draft manifest, not silently applied.
10
Confirm ingestion confirm-ingestion
Emits an intake-draft manifest; nothing ingested yet.
11
Propose refresh plan propose-refresh-plan
Layer-36 pipelines proposed for downstream invalidation.
12
Approve or skip refresh approve-or-skip-refresh
Reviewer authorization required for promotion; the console only proposes.
